Основы работы с Git

Что такое система контроля версий и зачем она нужна?

Ответ: Система контроля версий — это неотъемлемый инструмент для современных разработчиков. Она обеспечивает отслеживание изменений, совместную работу, управление версиями, защиту данных и позволяет эффективно вести разработку проектов.

Что делает команда git clone?

Ответ: Колонирует удаленный репозиторий

Что делает команда git init?

Ответ: Создает локальный репозиторий

Что такое коммит и для чего он используется?

Ответ: Скриншоты(история) изменений.

Как добавить изменения в индекс перед коммитом?

Ответ: git add

Для чего используется команда git checkout?

Ответ: Для переключения с ветки на ветку.

Для чего используется README файл в репозитории?

Ответ: Файл README (обычно README.md или README.txt) — это ключевой документ в репозитории проекта, который выполняет роль визитной карточки и руководства для тех, кто впервые сталкивается с вашим проектом. Он предназначен для предоставления информации о проекте, его особенностях, а также инструкций по его использованию, установке и вкладу.

Для чего используется файл .gitignore?

Ответ: Файл .gitignore — это текстовый файл, который используется в системах контроля версий (например, Git) для указания, какие файлы или папки должны быть игнорированы при отслеживании изменений в проекте. То есть, Git не будет включать эти файлы в свои коммиты, не будет отслеживать их изменения и не будет добавлять их в репозиторий.

Что такое SSH ключи и зачем они нужны?

Ответ: SSH-ключи (Secure Shell keys) — это механизм аутентификации, используемый для установления защищённого соединения между двумя компьютерами, например, между вашим компьютером и удалённым сервером. Они работают на основе асимметричного шифрования, предоставляя более безопасную альтернативу аутентификации по паролю.

Что такое GitHub Pages и как его использовать?

Ответ: GitHub Pages — это часть платформы GitHub, заточенная специально под размещение сайтов в интернете.
